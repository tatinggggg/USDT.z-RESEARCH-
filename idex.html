What is USDT.z — and why it isn't real

USDT.z is marketed as "Tether USD Bridged ZED20" — a bridged version of real USDT that lives on the ZEDXION chain. The implication is that for every USDT.z token in existence, a real USDT is locked somewhere as collateral. This is false.

Real bridged tokens work like this: you deposit 100 USDT on Ethereum → a bridge contract locks it → 100 wrapped tokens are minted on the destination chain. The lock and the mint are cryptographically linked. Examples: WBTC, axlUSDC, Stargate USDT.

USDT.z does not work this way. There is no locked USDT anywhere. The tokens were created from nothing using a template contract, with the entire supply minted to the deployer's own wallet.

The token contract — TeamToken, not a bridge

The USDT.z BSC contract's source code is verified on BscScan. The contract name is TeamToken — a generic token template distributed by Team.Finance, a token creation service. Anyone can deploy one for a few dollars.

Here is the actual constructor that created all 27.5 billion tokens:

// Contract name: TeamToken
// Deployed via Team.Finance token creation service
// Compiler: v0.6.12, Optimization: Yes

contract TeamToken is ERC20 {
    event TeamFinanceTokenMint(address owner);

    constructor(
        string memory name,       // "Tether USD"
        string memory symbol,     // "USDT.z"
        uint8 decimals,            // 18
        uint256 supply,            // 27,500,000,000 * 1e18
        address owner,            // their treasury wallet
        address feeWallet          // Team.Finance fee recipient
    ) {
        _setupDecimals(decimals);
        _mint(owner, supply);       // ALL tokens minted to deployer
        emit TeamFinanceTokenMint(owner);
    }
}
Critical finding There is no bridge logic. No locking mechanism. No validator. No proof of reserve. The entire 27.5 billion token supply was minted in a single transaction to their own wallet. This is identical to any meme coin deployment — the only difference is the name says "Tether USD."

04

The fake bridge — how "bridging" actually works

When you "bridge" USDT to USDT.z using their bridge at bridge.zedxion.com, here is what actually happens vs. what they claim:

What they claim
Your USDT is locked in a secure bridge contract. An equivalent USDT.z is minted on the ZED20 chain, backed 1:1 by your locked USDT.
What actually happens
Your USDT is sent to the treasury wallet 0xa3f37d00...779a6b. That wallet then sends you USDT.z from its 13.93B token reserve. There is no lock. There is no mint. Your real USDT goes into their pocket.
When you try to bridge back
You send USDT.z back to the treasury. They are supposed to send you USDT. Whether they do depends entirely on whether they choose to. There is no smart contract enforcing this. No code protects you. It is pure trust in an anonymous team.
A real bridge for comparison (Stargate / Omnibridge)
A smart contract automatically locks your deposit. A separate smart contract on the destination chain mints the wrapped token. Redemption is trustless — the lock and burn are atomic and enforced by code. No human can intercept your funds.

Wash trading — manufacturing fake volume

The PancakeSwap pair for USDT.z / USDT has approximately $2,445 in total liquidity yet processes around 880 transactions per day. This ratio is physically impossible for organic trading.

To understand why: if a pool has $2,445 in liquidity, a realistic daily volume would be $200–$500 (roughly 10–20% of liquidity). Instead, the reported volume is many multiples higher. The only explanation is that the same wallets are swapping funds back and forth repeatedly — called wash trading.

How the wash trading works

// Wallets A, B, C are all controlled by Zedxion

while (true) {
  const amount = random(0.001, 0.05)          // tiny random amounts
  const wallet = wallets[nextIndex()]

  // alternates buy and sell to create fake bidirectional volume
  await router.swapExactTokensForTokens(
    amount, 0,
    [USDT, USDT_Z],                            // or [USDT_Z, USDT] on next cycle
    wallet.address,
    deadline
  )

  await sleep(random(30, 180))               // random delay to look organic
}

The goal is not profit from trading — the gas fees eat any gain on tiny swaps. The goal is to generate transaction count and volume numbers that make analytics platforms like GeckoTerminal, Dexscreener, and CoinGecko report high activity, creating social proof for potential investors.

How to verify this yourself Go to BscScan, look up the PancakeSwap pair 0x1114e352349076d4159e7486a2430cd0d47b226b, click "Transactions" and examine the wallet addresses making swaps. You will see the same small group of wallets repeatedly swapping tiny amounts back and forth.

The private chain — ZEDX price is self-declared

The Zedxion explorer (ZedScan) shows ZEDX coin trading at $1,807.68 — a price higher than Ethereum. This figure appears nowhere on any independent exchange. No major CEX lists ZEDX. No reputable price aggregator independently verifies it.

Because ZEDXION is a private, permissioned blockchain controlled by the Zedxion team, they can display any price they want on their own explorer. There is no external validation. The $1,807 figure exists solely on zedscan.net — a website they run.

The circular value illusion Zedxion claims: USDT.z is backed by the ZED20 network. The ZED20 network has billions in assets. Those assets include USDT.z reserves. This is circular — USDT.z is backed by a system whose value is itself composed of USDT.z. Neither asset has independent external backing.

The DPoS validator claim

They claim a "Delegated Proof of Stake" consensus with validators. On a public blockchain like BSC or Ethereum you can verify validator sets on-chain. On their private chain, the validator list is whatever they say it is. There is no way to independently verify that these validators exist, are independent, or are not all controlled by the same team.

Final verdict

Zedxion / USDT.z is a manufactured legitimacy scheme. It is not technically a honeypot (you can sell the token) and it is not technically a rugpull in the classic sense. It is something more sophisticated: a fake ecosystem designed to make people believe they are holding a real stablecoin within a real blockchain economy.

The mechanics are:

  1. Print 27.5B tokens for free using a template contract
  2. Call them "bridged USDT" to borrow Tether's legitimacy
  3. Create a private blockchain where you control all prices and activity
  4. Run bots on real DEXes to fake trading volume and holder counts
  5. Submit your own price feeds to CMC / CoinGecko as the only source
  6. Build a neobanking app (ZedPay) where users deposit real money for fake tokens
  7. Collect real USDT from "bridge" users with no contractual obligation to return it
If you are currently holding USDT.z Do not deposit more. The only exit is selling on PancakeSwap (expect ~$0.008 per token) or attempting to "bridge back" through their platform (which requires trusting them to manually send you real USDT). Neither is guaranteed. There is no smart contract protecting your funds.
